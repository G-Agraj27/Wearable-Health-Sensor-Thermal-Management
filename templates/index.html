Low Power Health Sensors Optimization

Existing Routing Protocol

Asynchronous Data Transfer Routing

This protocol utilizes a standard NetworkX based routing approach that focuses on connectivity but lacks energy awareness. It often results in:

  • Higher power consumption
  • Unbalanced energy usage leading to voids
  • Increased recovery delays
High Power

Proposed Routing Protocol

Cloud Coordinated ZigBee

Our novel approach integrates cloud coordination with ZigBee sleep scheduling to optimize:

  • Power efficiency via sleep scheduling
  • Energy void handling success rate
  • Reduced rerouting overhead
  • Minimal recovery delay
Low Power

Algorithm Details

The system uses a Cloud Coordinated approach where the central controller (Cloud) maintains the global state of the network (managed via NetworkX graphs in the backend). It calculates optimal paths using a custom weighted metric that combines distance and residual energy.

Path Finding: nx.astar_path(G, current_id, target_ch_id, weight='energy_cost')
